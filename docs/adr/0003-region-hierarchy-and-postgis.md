# ADR 0003: 지역 처리 — 계층 region 테이블 + PostGIS

## 상태 (Status)

Accepted (2026-07-01)

## 배경

UI는 장소를 **지역 단위로 묶어** 표시하고("서울 마포구", 지도상 지역 원, "코스 5개" 같은 집계), 시도 > 시군구 > 지구처럼 여러 depth로 묶어야 한다. 지역명·계층·집계를 일관되게 관리할 방법이 필요하다.

## 결정

**계층형 `region` 테이블을 두고 `place.region_id`로 참조한다.**

- `region(parent_id, depth)`로 계층 표현(시도/시군구/지구).
- 지도 원 표시를 위해 **경계 폴리곤 대신 `center`(중심좌표) + `radius_meters`(반경)** 를 둔다.
- 지역 판정/자동 지정은 **PostGIS 좌표로 보완**(선택): 경계 폴리곤을 확보하면 `ST_Contains`로 place의 region 자동 지정 가능.

## 결과

### 긍정적

- 지역 그룹·집계·필터가 일관되고, 계층 묶음이 자연스럽다.
- 중심좌표+반경으로 지도 원을 간단히 그린다.

### 부정적

- 지역 데이터를 구축·유지해야 한다.

### 중립적

- 경계 데이터 확보 시 PostGIS 자동 지정으로 확장 가능(현재는 수동/좌표 기반).

## 검토한 대안

### 대안 1 — place에 region depth 컬럼

`place`에 시도·시군구 문자열 컬럼. 단순하지만 값 중복·오타 위험, 계층·집계가 약해 선택하지 않음.

### 대안 2 — PostGIS 전용(테이블 없음)

좌표만으로 처리. 지역명을 알려면 행정경계 폴리곤을 로드해야 하고, 이름·계층 관리엔 결국 테이블이 필요해 단독으로는 선택하지 않음(보완재로 채택).
